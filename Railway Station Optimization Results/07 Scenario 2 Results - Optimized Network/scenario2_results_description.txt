TITLE
  Scenario 2: Optimized Station Subset Selection — Modified
  K-means Results Across All k/λ Configurations

OVERVIEW
  Full results of the Scenario 2 optimization sweep, testing
  k ∈ {15, 20, 25} stations with λ ∈ {0.0, 0.3, 0.5, 0.7, 1.0}
  redundancy penalty values using modified K-means clustering
  with a geographic penalty term.

HOW THEY WERE GENERATED
  Modified K-means algorithm with the following additions:
    1. Geographic penalty: stations within 80 km of each
       other incur a λ-scaled cost penalty.
    2. Zone balance constraint: minimum 1 station per
       geopolitical zone enforced post-clustering.
    3. Mandatory station: Lagos (Ikeja) always included.
  The objective function minimized:
    α × Network_Length + β × Avg_Distance + λ × Redundancy
  Network length computed via MST on selected stations only.
  All 9 k/λ combinations were evaluated.

  corrected_metrics.json was produced by stage2_issue_audit.py
  which identified and corrected two systematic errors in the
  original run:
    1. Network length was computed as sum of all pairwise
       distances instead of MST edges → corrected.
    2. Scenario 1 avg distance to station was non-zero →
       corrected to 0.0 km (all capitals are stations).

PURPOSE
  Provides the full parameter sweep results to support
  sensitivity analysis and configuration selection. The
  corrected_metrics.json file contains the authoritative
  values to use in all reporting and publication.

VIEWING ORDER

TITLE
  Scenario Constraint Definitions and Parameter Grid for
  Nigeria HSR Station Optimization

OVERVIEW
  Three JSON configuration files encoding the mathematical
  constraints, objective function parameters, and
  hyperparameter search space used in the optimization study.

HOW THEY WERE GENERATED
  scenario1_constraints.json
    Hand-specified constraint set for the complete-coverage
    baseline (Scenario 1): k=37 stations (one per capital),
    minimum spacing 80 km, max route time 4 hours, objective
    = minimize total network length.

  scenario2_constraints.json
    Constraint set for the optimized subset selection
    (Scenario 2): k ∈ {15, 20, 25}, minimum spacing 80 km,
    max distance to station 150 km, mandatory inclusion of
    Lagos, minimum 1 station per geopolitical zone,
    neighbour redundancy penalty enabled.

  parameter_grid.json
    Full hyperparameter sweep definition:
      k values      : [15, 20, 25]
      λ penalties   : [0.0, 0.3, 0.5, 0.7, 1.0]
      Weight configs: ['cost_focused', 'balanced', 'access_focused']
      Spacing opts  : [80, 100, 120] km
    Total combinations: 45

PURPOSE
  Provides full reproducibility of the optimization
  experiment. Any researcher can re-run Stage 2 by loading
  these constraint files and applying them to master_dataset
  + distance_matrix inputs.

VIEWING ORDER
  Dataset #5 — review before examining Stage 2 results.
  Read scenario constraints before the results they produced.

OBJECTIVE FUNCTION (Scenario 2)
  Minimize: α × Network_Length + β × Avg_Distance_to_Station
            + λ × Redundancy_Penalty
  Where:
    cost_focused  : α=0.7, β=0.3
    balanced      : α=0.5, β=0.5
    access_focused: α=0.3, β=0.7
